# 🔧 控制台日志修复

## ❌ 问题
控制台没有显示 "预加载 OpenSSL DLL" 的日志

## ✅ 原因
应用程序被编译为 Windows GUI 模式，不显示控制台窗口。

## 🔨 解决方案

已修改 `CMakeLists.txt`：

```cmake
# 修改前：
WIN32_EXECUTABLE TRUE  # GUI 模式，无控制台

# 修改后：
WIN32_EXECUTABLE FALSE  # 控制台模式，显示调试信息
```

## 🔍 如何截图或复制日志

### 方法 1: 复制文本
1. 在控制台窗口中右键
2. 选择"全选"
3. 按 Enter 键复制
4. 粘贴到文本文件中

### 方法 2: 重定向到文件
创建一个批处理文件 `run.bat`：
```bat
@echo off
DataUploadTool.exe > debug.log 2>&1
pause
```

运行 `run.bat`，日志会保存到 `debug.log` 文件中。

### 方法 3: 截图
使用 Windows 截图工具（Win+Shift+S）截取控制台窗口。
